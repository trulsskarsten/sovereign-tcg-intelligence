# Project Handoff: Sovereign TCG Intelligence

## Architecture Overview

Sovereign TCG Intelligence is a Shopify Embedded App built with Next.js 16. It follows the **Managed Installation** model with **Token Exchange** for secure authentication. 

- **Frontend**: A high-density "Sovereign Light Glass" bento dashboard using `react-grid-layout`.
- **Backend**: Next.js API Routes protected by a custom `withAuth` middleware that validates Shopify session tokens and extracts store context.
- **Data Persistence**: Supabase PostgreSQL with Row Level Security (RLS) ensuring each store can only access its own records.
- **Asynchronous Processing**: Uses QStash for cron-based market scraping and batch operations, ensuring scalability beyond Vercel's serverless timeouts.

## Database Schema (Supabase)

The project uses Supabase project `omxqbognenqzeeiazjzy` with the following 10 tables (all verified against live schema):

1. `stores`: One row per installed shop — domain, encrypted access token, plan tier, widget layout JSON.
2. `inventory`: Local cache of Shopify variants with WAC cost, current qty, ABC class, and sync timestamp.
3. `store_settings`: Per-store automation settings — floor margin, panic lock threshold, enable/disable flags.
4. `market_prices`: Prices ingested from market scrapers (Pokepris, Cardmarket) per inventory item and source.
5. `price_history`: Immutable log of every executed price change — old price, new price, source, timestamp.
6. `staged_updates`: Queue of pending price changes awaiting merchant approval (status: pending → approved → executed).
7. `recommendations`: Historical record of pricing suggestions generated by the recommendation engine.
8. `purchases`: Stock arrival records used to calculate WAC (Weighted Average Cost) per variant.
9. `audit_logs`: Governance trail for system actions — syncs, GDPR requests, bulk executions, errors.
10. `scaper_logs`: Per-job telemetry from web scrapers — source, run time, result count, error details.

## Key Operational Flows

1. **Install Flow**: Managed installation grant -> Token exchange -> Store registration in DB -> Programmatic webhook registration.
2. **Inventory Sync**: Cursor-based GraphQL pagination fetches all products from Shopify and upserts them into the local `inventory` table.
3. **Pricing Loop**:
   - **Scrape**: Scheduled jobs fetch prices from Pokepris.no, Pokevarsler, and Outland.
   - **Recommend**: The engine evaluates each product against competition and profit floors.
   - **Stage**: Suggestions land in the `staged_updates` queue.
   - **Approve/Execute**: Merchant approves changes in the UI, which are then pushed to Shopify via GraphQL.

## Pricing Engine Modules

The core intelligence resides in `src/lib/automation/`:
- `floor.ts`: Calculates the absolute minimum price based on WAC + margin + VAT.
- `tiers.ts`: Validates price changes to prevent "flash crashes" using Panic Lock thresholds.
- `spread-engine.ts`: Manages competitive gap analysis.
- `abc_engine.ts`: Classifies products based on sales velocity and value.
- `profit.ts`: VAT-aware margin calculations.
- `recommendations.ts`: Orchestrates all rules to generate actionable updates.

## Known Limitations & Future Work

- **Scraping**: Vercel's 50MB deployment limit restricts large browser-based scrapers. Current implementation uses Cheerio for SSR parsing. For dynamic sites, consider a proxy or external scraping service.
- **Zod Validation**: Most public-facing API routes have full Zod schemas. Some internal administration routes currently rely solely on the `withAuth` middleware.
- **Upstash Redis**: Rate limiting and QStash signature verification are implemented but require real credentials to be active.
